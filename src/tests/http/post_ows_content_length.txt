#
#  fr_http_length() must strip trailing OWS from the Content-Length value
#  before parsing it (RFC 9112 §5.1).  Without the fix, strtoul stops at
#  the trailing space and returns non-numeric, causing fr_http_length() to
#  return -1 and close the connection before fr_http_decode() is reached.
#
#  Request (raw bytes):
#    POST /hello HTTP/1.1\r\n
#    Content-Length: 5 \r\n         <- trailing space after the digit
#    \r\n
#    hello
#
#  Expected: 201 (recv POST sets 201 for any path)
#
#  This test uses a raw TCP command rather than curl because curl normalises
#  Content-Length and cannot be made to send trailing whitespace in that header.
#  printf sends the exact bytes; head+cut+tr extract just the status code.
#
#  CMD: bash -c "exec 3<>/dev/tcp/127.0.0.1/PORT; printf '\x50\x4f\x53\x54\x20\x2f\x68\x65\x6c\x6c\x6f\x20\x48\x54\x54\x50\x2f\x31\x2e\x31\x0d\x0a\x43\x6f\x6e\x74\x65\x6e\x74\x2d\x4c\x65\x6e\x67\x74\x68\x3a\x20\x35\x20\x0d\x0a\x0d\x0a\x68\x65\x6c\x6c\x6f' >&3; head -1 <&3; exec 3>&-" | cut -d' ' -f2 | tr -d '\r\n'
